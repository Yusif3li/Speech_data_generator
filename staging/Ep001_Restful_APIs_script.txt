Speaker 2: سؤال ممتاز جداً يا سارة. الـ RESTful API هو طريقة معينة أو Architecture Style لتصميم الـ APIs دي، بتتبع مجموعة مبادئ وقواعد معينة عشان تكون سهلة الاستخدام، وفعالة، وقابلة للتوسع. كلمة REST اختصار لـ Representational State Transfer.

Speaker 1: "Representational State Transfer"... يا خبر أبيض! الاسم لوحده يخض! ممكن تبسطها أكتر؟ إيه المبادئ دي؟

Speaker 2: حاضر، هبسطها قدر الإمكان. تخيلي معايا إن أهم مبدأ في الـ RESTful API إن كل حاجة بنتعامل معاها هي "مورد" أو "Resource". والمورد ده بيكون ليه Unique Identifier، عادةً بيكون URL أو عنوان إنترنت. يعني مثلاً، لو عندك موقع فيه مستخدمين، ممكن يكون الـ Resource بتاعك هو "المستخدمين" أو "Users"، وكل مستخدم فردي ليه URL خاص بيه.

Speaker 1: زي لما أكتب مثلاً "facebook.com/myprofile"؟

Speaker 2: بالظبط كده، شبيه بده. والميزة الأساسية للـ RESTful APIs إنها بتستخدم الـ HTTP Methods أو أفعال بروتوكول الـ HTTP عشان تتفاعل مع الـ Resources دي. أشهر الـ Methods دي هما:
*   **GET:** بنستخدمه عشان نطلب بيانات من السيرفر. زي لما تفتحي بروفايل حد وتشوفي بوستاته.
*   **POST:** بنستخدمه عشان نبعت بيانات جديدة للسيرفر. زي لما تعملي بوست جديد أو تسجلي حساب جديد.
*   **PUT:** بنستخدمه عشان نعدل بيانات موجودة بالفعل. زي لما تعدلي بياناتك الشخصية في بروفايلك.
*   **DELETE:** وده واضح من اسمه، بنستخدمه عشان نحذف مورد أو بيانات معينة. زي لما تمسحي بوست قديم.
